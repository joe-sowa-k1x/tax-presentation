Tax Package Update 2027

Distinction of Roles & Jira Workflow

Manager Analyst E-File Analyst
📋

Three core roles

Each role owns a distinct layer of the process

Manager

Coordination, documentation intake, Jira readiness & final review

Analyst

Form analysis, PDF remapping & form mapping documentation

E-File Analyst

Schema analysis, e-file mapping, testing & state submissions

Manager

Primary ownership: intake, coordination & final review

Responsibilities
Manage LOIs and state communications
Handle and track FTA / SES documents
Pull documentation into Jira tickets and Form Filing Pod
Confirm source documentation is complete before analyst work
Review completed analyst work for quality and completeness
Communicate feedback; make minor tax-analysis updates directly
Determine when work is ready to proceed downstream
Deliverables
Updated Jira availability/status tracking
Complete source documentation in tickets and shared locations
Reviewed analyst work with feedback resolved
Clean handoff package for downstream teams

Analyst

Primary ownership: form-based workflow, templates & PDF mapping

Responsibilities
Complete the developer template based on the form
Review prior-year form/package vs. current year
Perform tax analysis for form changes
Identify line, field, layout, calculation & barcode changes
Re-map PDFs based on prior-year mappings and current-year changes
Update or create the mapping document in Excel
Incorporate manager review feedback and re-complete work as needed
Deliverables
Completed template for development
Updated PDF mapping
Tax analysis of form changes
Updated Excel mapping document

E-File Analyst

Primary ownership: schema analysis, e-file testing & state submissions

Responsibilities
Complete the developer template based on e-file schemas
Perform tax analysis of schema changes and e-file requirements
Identify and communicate schema mapping changes to development
Create/update Excel mapping document for new fields
Complete e-file test case creation based on 990Tracker requirements
Submit e-file packages to the state when applicable
Track, review, and communicate rejections
Deliverables
Completed schema-based template
Schema tax analysis
Updated Excel mapping document where needed
E-file test cases
Submission/rejection communication and tracking

Jira columns: Manager-owned

Stages owned by the Manager

Manager

To Do

Forms identified for the current cycle. Manager creates tickets and establishes the initial queue.

LOI Available

State provides the LOI. Determines supported/required forms. Update initial and final deadlines.

LOI Submitted

LOI completed and submitted to the state for intended form support.

LOI Approved

State approves the LOI. SES access granted for receiving forms and documentation.

Docs Pulled

Latest forms pulled from SES. Documents attached to Jira ticket. Received dates updated.

Jira columns: Analyst & E-File Analyst-owned

Stages owned by Analyst and E-File Analyst

Analyst / E-File

Template Complete

Ticket completed with description, Draftable link, new form and continuous form indicators.

Tax Analysis

Compare current vs. prior year. Document all changes — lines, fields, calculations, layout, schema.

Format Analysis

Update PDF from prior-year baseline. Validate alignment. Upload to Form Library; update Excel mapping.

Mapping Document

Update state mapping Excel. Includes calculation updates, field sourcing, and new/changed mapping logic.

Test Case Creation

Create test cases from state requirements, prior submissions, and the internal log. Cover all scenarios.

Ready to send to jurisdiction

Manager
All analysis, mapping, formatting, and testing is complete
Work item is finalized and ready for submission to the state jurisdiction
State returns with feedback, included in the ticket and moved to its respective column

Final gate before submission

This column closes out Analyst and E-File Analyst work and hands off to the jurisdiction.
